## 1. 引言

### 1.1 目的

本文件為進銷存管理系統（Inventory Management System，以下簡稱「本系統」）的軟體需求規格書，旨在定義系統的功能性與非功能性需求，作為後續設計、開發、測試及驗收之依據。

### 1.2 範圍

本系統涵蓋企業日常進貨、銷貨、庫存管理之完整作業流程，並包含財務整合、多通路同步、報表分析及進階倉儲功能。系統定位為中小型至中大型企業使用的雲端進銷存平台。

### 1.3 定義與縮寫

| 術語 | 說明 |
|------|------|
| SRS | Software Requirements Specification，軟體需求規格書 |
| SKU | Stock Keeping Unit，庫存單位識別碼 |
| PO | Purchase Order，採購單 |
| SO | Sales Order，銷售訂單 |
| BOM | Bill of Materials，物料清單 |
| FIFO | First In First Out，先進先出 |
| FEFO | First Expired First Out，先到期先出 |
| EOQ | Economic Order Quantity，經濟訂購量 |
| RMA | Return Merchandise Authorization，退貨授權 |
| WMS | Warehouse Management System，倉庫管理系統 |
| 3PL | Third-Party Logistics，第三方物流 |
| ABC分析 | 依銷售額/頻率將品項分為A、B、C三類 |

## 2. 整體描述

### 2.1 產品概覽

本系統為一套基於雲端架構的進銷存管理平台，支援多使用者、多倉庫、多幣別、多通路操作。系統採用模組化設計，企業可依需求啟用或停用特定模組。

### 2.2 產品功能摘要

```
進銷存管理系統
├── 商品管理          ← 品項主檔、分類、屬性、BOM
├── 採購管理          ← 詢報價、採購單、進貨驗收、退貨
├── 銷售管理          ← 報價單、銷售訂單、出貨、退貨
├── 庫存管理          ← 即時庫存、調撥、盤點、成本
├── 倉庫與倉位管理    ← 多倉庫、倉位、揀貨路徑
├── 財務整合          ← 應收應付、多幣別、會計分錄
├── 報表與分析        ← 進銷存報表、ABC分析、預測
├── 自動化            ← 工作流程、通知、排程作業
├── 系統整合          ← ERP、電商平台、物流、支付
└── 進階功能          ← 寄銷、委外、效期、保固追蹤
```

### 2.3 使用者與市場

- **主要使用族群**：中小型製造業、批發商、電商業者、零售商
- **操作環境**：Web 瀏覽器（桌面）、行動裝置（iOS / Android App）
- **部署方式**：雲端 SaaS（主要）、私有雲（選配）

### 2.4 設計假設與限制

- 系統須支援最少 100 位並發使用者
- 單一帳號下商品品項上限不少於 100,000 筆
- 所有核心交易資料須支援稽核日誌
- 日期時間以 UTC 儲存，前端依時區轉換顯示

### 4.1 商品管理（Item Management）

#### 4.1.1 商品主檔管理

**功能描述**：系統提供完整的商品資料維護介面，支援多層分類、多屬性管理。

**功能需求**：

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| IM-001 | 建立商品 | 可輸入商品名稱、描述、圖片（多張）、分類、單位、稅別 |
| IM-002 | 商品分類管理 | 支援多層級分類樹狀結構（最多 5 層） |
| IM-003 | 多屬性變體 | 支援顏色、尺寸、規格等屬性，自動展開為 SKU 組合 |
| IM-004 | 多單位換算 | 同一商品可定義多種計量單位（如：個、打、箱），系統自動換算 |
| IM-005 | 商品停用/刪除 | 停用後不出現在新單據選項；有交易紀錄者只可停用不可刪除 |
| IM-006 | 商品複製 | 以現有商品為範本快速建立新商品 |
| IM-007 | 批次匯入/匯出 | 支援 CSV / Excel 批次匯入商品主檔；支援匯出 |

#### 4.1.2 SKU 管理

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| IM-010 | SKU 自動產生 | 可設定 SKU 編碼規則（前綴、流水號、屬性碼組合），系統自動產生 |
| IM-011 | SKU 唯一性驗證 | 系統確保同帳號下 SKU 不重複 |
| IM-012 | 條碼綁定 | 一個 SKU 可綁定多組條碼（Barcode / QR Code / EAN / UPC） |

#### 4.1.3 商品組合（BOM / Kit）

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| IM-020 | 套件商品 (Kit) | 定義由多個子品項組合而成的套件，銷售時自動扣除各子品庫存 |
| IM-021 | 組合生產 (Assembly) | 支援製造 BOM，記錄原物料消耗與半成品/成品入庫 |
| IM-022 | 多層 BOM | 支援 BOM 巢狀結構（原物料 → 半成品 → 成品） |

#### 4.1.4 商品定價

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| IM-030 | 多價格表 | 可建立多組價格表（如：零售價、批發價、VIP 價、外幣價） |
| IM-031 | 階梯定價 | 依數量區間設定不同單價（如：買 1–10 個 $100；買 11+ 個 $90） |
| IM-032 | 有效期價格 | 價格表可設定起訖日期，過期自動回復標準售價 |

| PO-002 | 供應商商品目錄 | 記錄每位供應商可供應的品項及其報價、前置時間（Lead Time） |
| PO-003 | 供應商評等 | 可記錄供應商品質評分、準時交貨率；支援備註 |
| PO-004 | 供應商停用 | 停用後無法選為新採購單的供應商 |

#### 4.2.2 詢報價作業

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| PO-010 | 詢價單 (RFQ) | 建立詢價單並發送至多位供應商 |
| PO-011 | 供應商報價比較 | 彙整多家供應商報價，以表格並排進行比較 |
| PO-012 | 詢價轉採購單 | 選定供應商後一鍵轉為採購單 |

#### 4.2.3 採購單作業

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|

| PO-024 | 採購單修改紀錄 | 採購單核准後修改須留存修改原因與版本歷程 |

#### 4.2.4 進貨驗收

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|

| PO-031 | 條碼掃描入庫 | 支援手持掃描器/手機進行條碼掃描入庫 |
| PO-032 | 批號 / 序號登錄 | 到貨時輸入批號、序號、生產日期、到期日 |

#### 4.2.5 採購退貨

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| PO-040 | 採購退貨單 | 建立退貨申請，記錄退貨原因、數量 |
| PO-041 | 退貨出庫 | 確認退貨後自動扣減庫存 |
| PO-042 | 供應商退款/折讓 | 記錄供應商退款金額或開立折讓單抵帳 |

#### 4.2.6 採購付款

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|

| PO-051 | 付款排程 | 可設定付款到期日，到期前自動提醒財務人員 |

### 4.4 庫存管理（Inventory Management）

#### 4.4.1 庫存查詢

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| INV-001 | 即時庫存查詢 | 可依商品、倉庫、倉位查詢即時在庫量、可用量、預留量、在途量 |
| INV-002 | 庫存異動明細 | 查看某商品所有入庫/出庫/調撥的完整歷程 |
| INV-003 | 批號/序號查詢 | 依批號或序號查詢其入庫、存放位置、出庫去向 |
| INV-004 | 負庫存允許設定 | 系統管理員可設定特定品項是否允許負庫存（先出貨再補） |

#### 4.4.2 庫存調撥

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| INV-010 | 倉庫間調撥 | 建立庫存調撥單，扣減來源倉庫並增加目的倉庫庫存 |
| INV-011 | 倉位間移動 | 同倉庫內不同倉位間移動庫存 |
| INV-012 | 在途庫存追蹤 | 調撥確認後至到達前，顯示「在途」狀態庫存 |

#### 4.4.3 庫存盤點

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| INV-020 | 全盤 | 建立全倉庫盤點任務，列出所有品項 |
| INV-021 | 循環盤點 | 依 ABC 分類設定盤點頻率（A類每週、B類每月、C類每季） |
| INV-022 | 盤點凍結 / 快照 | 盤點期間凍結庫存快照，盤點仍可繼續入出貨，結束後自動計算差異 |
| INV-023 | 盤盈盤虧記錄 | 盤點完成後產生盈虧明細，待審核確認後更新正式庫存 |
| INV-024 | 行動裝置盤點 | 支援用手機/平板掃描條碼進行盤點作業 |

#### 4.4.4 庫存成本

| 需求編號 | 功能名稱 | 說明 |
|----------|----------|------|
| INV-030 | 成本計算方式設定 | 支援加權平均成本法（WAC）、先進先出（FIFO） |
| INV-031 | 到岸成本分攤 (Landed Cost) | 進口商品可加入運費、關稅、保險費，分攤至各品項取得真實成本 |
| INV-032 | 成本重算 | 手動触發重算庫存成本（如：修正到岸成本後） |

### 4.6 財務與帳款管理（Finance & Billing）

| 需求編號 | 功能名稱 | 說明 |

## 6. 資料模型概要

### 6.1 核心實體關係

```
[商品主檔 Item]
    ├── [SKU]
    │     ├── [條碼 Barcode] (多)
    │     └── [商品屬性 Attribute]
    └── [BOM / Kit 結構]

[採購單 PurchaseOrder]
    ├── [供應商 Supplier]
    ├── [採購明細 PurchaseOrderLine] → [SKU]
    └── [進貨單 GoodsReceipt]
          ├── [進貨明細] → [SKU]
          └── [批號/序號 Batch/Serial]

[銷售訂單 SalesOrder]
    ├── [客戶 Customer]
    ├── [訂單明細 SalesOrderLine] → [SKU]
    └── [出貨單 Shipment]
          ├── [出貨明細] → [SKU] → [批號/序號]
          └── [物流資訊 ShippingInfo]

[庫存 Inventory]
    ├── [倉庫 Warehouse]
    │     └── [倉位 Bin]
    ├── [SKU]
    ├── [批號/序號 Batch/Serial]
    └── [庫存異動日誌 InventoryTransaction]

[發票 Invoice] → [銷售訂單]
[帳單 Bill] → [採購單]
[付款/收款 Payment] → [發票/帳單]
```

### 6.2 關鍵欄位說明

#### 庫存 (Inventory) 核心欄位

| 欄位 | 說明 |
|------|------|
| on_hand_qty | 實際在庫數量 |
| reserved_qty | 已被銷售訂單預留的數量 |
| available_qty | 可用數量（= on_hand - reserved） |
| on_order_qty | 已下採購單但尚未到貨的數量（在途） |
| reorder_point | 再訂購點數量 |
| safety_stock | 安全庫存量 |

#### 庫存異動日誌 (InventoryTransaction) 關鍵欄位

| 欄位 | 說明 |
|------|------|
| transaction_type | 類型（IN/OUT/TRANSFER/ADJUST/COUNT） |
| source_document_type | 來源單據類型（PO/SO/TRANSFER/ADJUSTMENT） |
| source_document_id | 來源單據 ID |
| quantity | 異動數量（正為入庫，負為出庫） |
| unit_cost | 該筆交易的單位成本 |
| performed_by | 操作人員 ID |
| performed_at | 操作時間（UTC） |

## 10. 詞彙表

| 術語 | 中文說明 |
|------|----------|
| 安全庫存 (Safety Stock) | 緩衝庫存量，防止需求突增或供應延遲時缺貨 |
| 再訂購點 (Reorder Point) | 庫存降至此數量時觸發補貨 |
| 前置時間 (Lead Time) | 從下採購單到貨物到達所需天數 |
| 庫存週轉率 (Inventory Turnover) | 一定期間內庫存周轉次數，反映庫存效率 |
| 呆滯料 (Dead Stock / Slow-Moving) | 長期無銷售異動的庫存，佔據倉儲空間與資金 |
| 到岸成本 (Landed Cost) | 進口商品的 CIF 費用，含商品成本+運費+關稅+保險 |
| 寄銷 (Consignment) | 貨物暫存於客戶/通路端，賣出後才計銷售 |
| Drop Shipping | 訂單由供應商直接出貨給客戶，無需經過賣方倉庫 |
| 序號追蹤 (Serial Number Tracking) | 逐件追蹤每個商品的流向 |
| 批號追蹤 (Batch/Lot Tracking) | 以生產批次追蹤同批商品的流向 |
| FEFO | First Expired First Out，先到期先出 |
| FIFO | First In First Out，先進先出 |
| EOQ | Economic Order Quantity，經濟訂購量公式 |
| 保固 (Warranty) | 售出商品在保固期內的維修/換貨承諾 |
| RMA | Return Merchandise Authorization，退貨授權單 |
| 倉位 (Bin) | 倉庫內最小儲存單元（貨架格位） |
| 稽核軌跡 (Audit Trail) | 所有系統操作的完整且不可竄改的記錄 |
| 多通路 (Omni-Channel) | 同時透過多個銷售管道（實體/電商/批發）進行銷售 |
